## Key Achievements

### First CalDAV Server with Complete DeltaV Write Operations

As of January 2025, Radicale is the **first CalDAV server** to implement complete RFC 3253 write operations:

- ✅ **VERSION-CONTROL** - Place resources under version control
- ✅ **CHECKOUT** - Lock items for editing with fork policies
- ✅ **CHECKIN** - Create new versions and unlock
- ✅ **UNCHECKOUT** - Cancel checkouts and discard changes
- ✅ **LABEL** - Add/move/remove version labels (git tags)
- ✅ **MKACTIVITY** - Create activities for change set grouping
- ✅ **VERSION-TREE REPORT** - Query version history
- ✅ **Auto-versioning** - Transparent version tracking

[Learn more about versioning →](/rfc-compliance/3253-deltav/)

### Full Bidirectional Email Scheduling

Radicale implements complete **RFC 6047 iMIP** support, a rare feature in open-source CalDAV servers:

**Outbound (Organizer → External Attendee):**
- ✅ RFC 6047 compliant MIME structure
- ✅ HTML + text/calendar multipart
- ✅ Proper METHOD headers

**Inbound (External Attendee → Organizer):**
- ✅ IMAP polling (background thread)
- ✅ HTTP webhooks (SendGrid, Mailgun, Postmark, Generic)
- ✅ REPLY processing updates PARTSTAT
- ✅ Security validation (sender/ATTENDEE matching)

[Learn more about iMIP →](/rfc-compliance/6047-imip/)

### Near-Complete CalDAV Scheduling

Radicale's **RFC 6638** implementation rivals commercial servers:

- ✅ All iTIP methods: REQUEST, REPLY, CANCEL, COUNTER, DECLINECOUNTER, REFRESH
- ✅ Internal routing for same-server attendees
- ✅ SCHEDULE-AGENT=SERVER for resource calendars
- ✅ Free/busy queries with availability
- ✅ Delegation support (DELEGATED-TO/DELEGATED-FROM)
- ✅ Recurring event scheduling with RECURRENCE-ID
- ✅ 48+ comprehensive tests

[Learn more about scheduling →](/rfc-compliance/6638-scheduling/)

## Known Limitations

<Aside type="caution">
While Radicale has excellent RFC compliance, there are some intentional gaps:

**WebDAV LOCK/UNLOCK**: Not implemented (most CalDAV clients don't require it)

**RFC 3253 Advanced Features**: MERGE, BASELINE, WORKSPACE not implemented (rarely used)

**CalDAV Proxy**: Delegation works through scheduling, not full CalDAV proxy protocol
</Aside>
