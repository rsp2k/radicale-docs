## Overview

RFC 6352 defines **vCard Extensions to WebDAV (CardDAV)**, enabling contact management through the WebDAV protocol. Radicale implements the core CardDAV specification with excellent compliance, supporting addressbook creation, contact queries, and synchronization.

**Compliance Level**: ~90% (core features complete)

## Key Features

### Create Addressbook

Create addressbook collections:

```bash
# Create addressbook
curl -X MKCOL \
  -u alice:password \
  -H "Content-Type: application/xml" \
  -d '<?xml version="1.0" encoding="utf-8"?>
      <D:mkcol xmlns:D="DAV:" xmlns:C="urn:ietf:params:xml:ns:carddav">
        <D:set>
          <D:prop>
            <D:resourcetype>
              <D:collection/>
              <C:addressbook/>
            </D:resourcetype>
            <D:displayname>Personal Contacts</D:displayname>
            <C:addressbook-description>My personal contacts</C:addressbook-description>
          </D:prop>
        </D:set>
      </D:mkcol>' \
  https://server/alice/contacts.vcf/
```

### Store vCard Contact

Add contacts using PUT:

```bash
# Store vCard 4.0 contact
curl -X PUT \
  -u alice:password \
  -H "Content-Type: text/vcard" \
  -d 'BEGIN:VCARD
VERSION:4.0
FN:John Doe
N:Doe;John;;;
EMAIL:john@example.com
TEL;TYPE=work:+1-555-1234
ADR;TYPE=work:;;123 Main St;Springfield;IL;62701;USA
ORG:Example Corp
TITLE:Software Engineer
UID:john-doe-123
END:VCARD' \
  https://server/alice/contacts.vcf/john-doe.vcf
```

### addressbook-query REPORT

Search contacts with filters:

```bash
# Find contacts by name
curl -X REPORT \
  -u alice:password \
  -H "Content-Type: application/xml" \
  -d '<?xml version="1.0" encoding="utf-8"?>
      <C:addressbook-query xmlns:D="DAV:" xmlns:C="urn:ietf:params:xml:ns:carddav">
        <D:prop>
          <D:getetag/>
          <C:address-data/>
        </D:prop>
        <C:filter>
          <C:prop-filter name="FN">
            <C:text-match collation="i;unicode-casemap">John</C:text-match>
          </C:prop-filter>
        </C:filter>
      </C:addressbook-query>' \
  https://server/alice/contacts.vcf/
```

### addressbook-multiget REPORT

Retrieve multiple contacts efficiently:

```bash
# Get multiple contacts by href
curl -X REPORT \
  -u alice:password \
  -H "Content-Type: application/xml" \
  -d '<?xml version="1.0" encoding="utf-8"?>
      <C:addressbook-multiget xmlns:D="DAV:" xmlns:C="urn:ietf:params:xml:ns:carddav">
        <D:prop>
          <D:getetag/>
          <C:address-data/>
        </D:prop>
        <D:href>/alice/contacts.vcf/john-doe.vcf</D:href>
        <D:href>/alice/contacts.vcf/jane-smith.vcf</D:href>
        <D:href>/alice/contacts.vcf/bob-jones.vcf</D:href>
      </C:addressbook-multiget>' \
  https://server/alice/contacts.vcf/
```

## Addressbook Properties

PROPFIND returns CardDAV-specific properties:

| Property | Description |
|----------|-------------|
| `addressbook-description` | Human-readable addressbook description |
| `supported-address-data` | Supported vCard versions and media types |
| `max-resource-size` | Maximum size of vCard object |
| `principal-address` | Principal's preferred contact information |

### Query Properties Example

```bash
curl -X PROPFIND \
  -u alice:password \
  -H "Depth: 0" \
  -H "Content-Type: application/xml" \
  -d '<?xml version="1.0" encoding="utf-8"?>
      <D:propfind xmlns:D="DAV:" xmlns:C="urn:ietf:params:xml:ns:carddav">
        <D:prop>
          <D:displayname/>
          <C:addressbook-description/>
          <C:supported-address-data/>
        </D:prop>
      </D:propfind>' \
  https://server/alice/contacts.vcf/
```

## Advanced Filtering

### Property Filters

Filter by vCard properties:

```xml
<!-- Search by email -->
<C:filter>
  <C:prop-filter name="EMAIL">
    <C:text-match>@example.com</C:text-match>
  </C:prop-filter>
</C:filter>
```

```xml
<!-- Search by organization -->
<C:filter>
  <C:prop-filter name="ORG">
    <C:text-match>Example Corp</C:text-match>
  </C:prop-filter>
</C:filter>
```

```xml
<!-- Search by phone number -->
<C:filter>
  <C:prop-filter name="TEL">
    <C:text-match>555</C:text-match>
  </C:prop-filter>
</C:filter>
```

### Multiple Filters

Combine multiple property filters:

```xml
<C:filter>
  <!-- Contacts in Example Corp with "engineer" in title -->
  <C:prop-filter name="ORG">
    <C:text-match>Example Corp</C:text-match>
  </C:prop-filter>
  <C:prop-filter name="TITLE">
    <C:text-match collation="i;unicode-casemap">engineer</C:text-match>
  </C:prop-filter>
</C:filter>
```

### Case-Insensitive Search

Use collation for case-insensitive matching:

```xml
<C:prop-filter name="FN">
  <C:text-match collation="i;unicode-casemap">john</C:text-match>
</C:prop-filter>
```

## vCard Support

### vCard 3.0 (RFC 2426)

```vcard
BEGIN:VCARD
VERSION:3.0
FN:John Doe
N:Doe;John;;;
EMAIL;TYPE=INTERNET:john@example.com
TEL;TYPE=WORK:+1-555-1234
ADR;TYPE=WORK:;;123 Main St;Springfield;IL;62701;USA
ORG:Example Corp
TITLE:Software Engineer
UID:john-doe-123
END:VCARD
```

### vCard 4.0 (RFC 6350) - Recommended

```vcard
BEGIN:VCARD
VERSION:4.0
FN:John Doe
N:Doe;John;;;
EMAIL:john@example.com
TEL;TYPE=work:+1-555-1234
ADR;TYPE=work:;;123 Main St;Springfield;IL;62701;USA
ORG:Example Corp
TITLE:Software Engineer
PHOTO:https://example.com/photos/john.jpg
UID:john-doe-123
END:VCARD
```

### Contact Groups (KIND=group)

```vcard
BEGIN:VCARD
VERSION:4.0
FN:Engineering Team
KIND:group
MEMBER:urn:uuid:john-doe-123
MEMBER:urn:uuid:jane-smith-456
MEMBER:urn:uuid:bob-jones-789
UID:eng-team-group
END:VCARD
```

## LDAP Directory Gateway

Radicale can expose LDAP directories as read-only CardDAV addressbooks:

```ini
[storage]
type = multifilesystem

[auth]
type = ldap

# LDAP directory gateway configuration
ldap_directory_gateway = True
ldap_directory_path = /directory/
ldap_base_dn = ou=people,dc=example,dc=com
ldap_filter = (objectClass=person)
```

Clients can then access LDAP entries via CardDAV:

```bash
# Query LDAP directory via CardDAV
curl -X REPORT \
  -u alice:password \
  -H "Content-Type: application/xml" \
  -d '<?xml version="1.0" encoding="utf-8"?>
      <C:addressbook-query xmlns:D="DAV:" xmlns:C="urn:ietf:params:xml:ns:carddav">
        <D:prop><C:address-data/></D:prop>
        <C:filter>
          <C:prop-filter name="FN">
            <C:text-match>smith</C:text-match>
          </C:prop-filter>
        </C:filter>
      </C:addressbook-query>' \
  https://server/directory/
```

LDAP attributes are automatically mapped to vCard properties.

## Synchronization

### Collection Change Token (CTag)

```bash
# Get addressbook ctag
curl -X PROPFIND \
  -u alice:password \
  -H "Depth: 0" \
  -H "Content-Type: application/xml" \
  -d '<?xml version="1.0" encoding="utf-8"?>
      <D:propfind xmlns:D="DAV:" xmlns:CS="http://calendarserver.org/ns/">
        <D:prop>
          <CS:getctag/>
        </D:prop>
      </D:propfind>' \
  https://server/alice/contacts.vcf/
```

### Resource ETags

```bash
# Get contact ETag
curl -I -u alice:password \
  https://server/alice/contacts.vcf/john-doe.vcf

# Use ETag for conditional update
curl -X PUT \
  -u alice:password \
  -H "If-Match: \"abc123\"" \
  -H "Content-Type: text/vcard" \
  -d @updated-contact.vcf \
  https://server/alice/contacts.vcf/john-doe.vcf
```

### sync-collection REPORT (RFC 6578)

```bash
# Efficient incremental sync
curl -X REPORT \
  -u alice:password \
  -H "Content-Type: application/xml" \
  -d '<?xml version="1.0" encoding="utf-8"?>
      <D:sync-collection xmlns:D="DAV:">
        <D:sync-token/>
        <D:sync-level>1</D:sync-level>
        <D:prop>
          <D:getetag/>
        </D:prop>
      </D:sync-collection>' \
  https://server/alice/contacts.vcf/
```
