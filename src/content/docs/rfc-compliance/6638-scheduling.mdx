## Overview

RFC 6638 defines **CalDAV Scheduling Extensions**, enabling automated meeting invitations, responses, and coordination between calendar users. Radicale implements near-complete scheduling support, including all iTIP methods, internal routing, delegation, and resource calendars.

**Compliance Level**: ~95% (missing only edge cases)

## Key Features

### Meeting Invitation Flow

**Organizer creates meeting and invites attendees:**

```bash
# 1. Organizer creates event with attendees
curl -X PUT \
  -u alice:password \
  -H "Content-Type: text/calendar" \
  -d 'BEGIN:VCALENDAR
VERSION:2.0
PRODID:-//Radicale//EN
BEGIN:VEVENT
UID:meeting-123@example.com
DTSTART:20250120T140000Z
DTEND:20250120T150000Z
SUMMARY:Team Meeting
ORGANIZER:mailto:alice@example.com
ATTENDEE;CN=Bob;RSVP=TRUE:mailto:bob@example.com
ATTENDEE;CN=Carol;RSVP=TRUE:mailto:carol@example.com
SEQUENCE:0
STATUS:CONFIRMED
END:VEVENT
END:VCALENDAR' \
  https://server/alice/calendar.ics/meeting-123.ics
```

**Radicale automatically:**
1. Delivers REQUEST to bob@example.com and carol@example.com
2. For internal users (same server), adds to their schedule-inbox
3. For external users, sends iMIP email (if configured)

### Attendee Response Flow

**Bob responds to invitation:**

```bash
# 2. Bob accepts invitation
curl -X PUT \
  -u bob:password \
  -H "Content-Type: text/calendar" \
  -d 'BEGIN:VCALENDAR
VERSION:2.0
PRODID:-//Radicale//EN
BEGIN:VEVENT
UID:meeting-123@example.com
DTSTART:20250120T140000Z
DTEND:20250120T150000Z
SUMMARY:Team Meeting
ORGANIZER:mailto:alice@example.com
ATTENDEE;CN=Bob;PARTSTAT=ACCEPTED:mailto:bob@example.com
SEQUENCE:0
END:VEVENT
END:VCALENDAR' \
  https://server/bob/calendar.ics/meeting-123.ics
```

**Radicale automatically:**
1. Sends REPLY to alice@example.com
2. Updates PARTSTAT in Alice's copy to ACCEPTED
3. Delivers reply to Alice's schedule-inbox

### Schedule-Outbox POST

Explicitly send scheduling messages via POST:

```bash
# Send meeting invitation via outbox
curl -X POST \
  -u alice:password \
  -H "Content-Type: text/calendar; charset=utf-8" \
  -d 'BEGIN:VCALENDAR
VERSION:2.0
METHOD:REQUEST
PRODID:-//Radicale//EN
BEGIN:VEVENT
UID:meeting-456@example.com
DTSTART:20250121T100000Z
DTEND:20250121T110000Z
SUMMARY:Project Review
ORGANIZER:mailto:alice@example.com
ATTENDEE;CN=Bob;RSVP=TRUE:mailto:bob@example.com
SEQUENCE:0
END:VEVENT
END:VCALENDAR' \
  https://server/alice/schedule-outbox/
```

Response includes delivery status for each recipient.

## Scheduling Properties

### Calendar User Properties

PROPFIND returns scheduling-specific properties:

| Property | Description |
|----------|-------------|
| `schedule-inbox-URL` | URL of user's schedule-inbox |
| `schedule-outbox-URL` | URL of user's schedule-outbox |
| `schedule-default-calendar-URL` | Default calendar for scheduled events |
| `calendar-user-address-set` | User's email addresses for scheduling |
| `calendar-user-type` | USER type (INDIVIDUAL, GROUP, RESOURCE, ROOM) |

```bash
curl -X PROPFIND \
  -u alice:password \
  -H "Depth: 0" \
  -H "Content-Type: application/xml" \
  -d '<?xml version="1.0"?>
      <D:propfind xmlns:D="DAV:" xmlns:C="urn:ietf:params:xml:ns:caldav">
        <D:prop>
          <C:schedule-inbox-URL/>
          <C:schedule-outbox-URL/>
          <C:schedule-default-calendar-URL/>
          <C:calendar-user-address-set/>
        </D:prop>
      </D:propfind>' \
  https://server/alice/
```

## iTIP Method Details

### REQUEST - Invitations

Organizer sends meeting invitation:

```icalendar
BEGIN:VCALENDAR
VERSION:2.0
METHOD:REQUEST
BEGIN:VEVENT
UID:meeting@example.com
DTSTART:20250120T140000Z
DTEND:20250120T150000Z
SUMMARY:Team Meeting
ORGANIZER:mailto:alice@example.com
ATTENDEE;RSVP=TRUE:mailto:bob@example.com
SEQUENCE:0
END:VEVENT
END:VCALENDAR
```

### REPLY - Responses

Attendee responds with participation status:

```icalendar
BEGIN:VCALENDAR
VERSION:2.0
METHOD:REPLY
BEGIN:VEVENT
UID:meeting@example.com
ORGANIZER:mailto:alice@example.com
ATTENDEE;PARTSTAT=ACCEPTED:mailto:bob@example.com
SEQUENCE:0
REQUEST-STATUS:2.0;Success
END:VEVENT
END:VCALENDAR
```

Supported PARTSTAT values:
- `NEEDS-ACTION` - No response yet
- `ACCEPTED` - Attendee accepted
- `DECLINED` - Attendee declined
- `TENTATIVE` - Tentatively accepted
- `DELEGATED` - Delegated to another user

### CANCEL - Cancellations

Organizer cancels meeting:

```icalendar
BEGIN:VCALENDAR
VERSION:2.0
METHOD:CANCEL
BEGIN:VEVENT
UID:meeting@example.com
DTSTART:20250120T140000Z
DTEND:20250120T150000Z
SUMMARY:Team Meeting
ORGANIZER:mailto:alice@example.com
ATTENDEE:mailto:bob@example.com
SEQUENCE:1
END:VEVENT
END:VCALENDAR
```

### COUNTER - Counter Proposals

Attendee proposes alternative time:

```icalendar
BEGIN:VCALENDAR
VERSION:2.0
METHOD:COUNTER
BEGIN:VEVENT
UID:meeting@example.com
DTSTART:20250120T150000Z
DTEND:20250120T160000Z
SUMMARY:Team Meeting
ORGANIZER:mailto:alice@example.com
ATTENDEE;PARTSTAT=TENTATIVE:mailto:bob@example.com
COMMENT:Can we push this one hour later?
SEQUENCE:0
END:VEVENT
END:VCALENDAR
```

### DECLINECOUNTER - Reject Counter

Organizer rejects counter proposal:

```icalendar
BEGIN:VCALENDAR
VERSION:2.0
METHOD:DECLINECOUNTER
BEGIN:VEVENT
UID:meeting@example.com
DTSTART:20250120T140000Z
DTEND:20250120T150000Z
ORGANIZER:mailto:alice@example.com
ATTENDEE:mailto:bob@example.com
SEQUENCE:0
END:VEVENT
END:VCALENDAR
```

### REFRESH - Request Update

Attendee requests current event state:

```icalendar
BEGIN:VCALENDAR
VERSION:2.0
METHOD:REFRESH
BEGIN:VEVENT
UID:meeting@example.com
ORGANIZER:mailto:alice@example.com
ATTENDEE:mailto:bob@example.com
END:VEVENT
END:VCALENDAR
```

## Advanced Features

### Resource Calendars

Configure auto-accept for rooms and equipment:

```ini
[scheduling]
# Enable scheduling
enabled = True

# Resource calendars auto-accept
resource_calendars = /rooms/, /equipment/
```

```bash
# Create room calendar
curl -X MKCALENDAR \
  -u admin:password \
  -H "Content-Type: application/xml" \
  -d '<?xml version="1.0"?>
      <C:mkcalendar xmlns:D="DAV:" xmlns:C="urn:ietf:params:xml:ns:caldav">
        <D:set>
          <D:prop>
            <D:displayname>Conference Room A</D:displayname>
            <C:calendar-user-type>ROOM</C:calendar-user-type>
            <C:schedule-agent>SERVER</C:schedule-agent>
          </D:prop>
        </D:set>
      </C:mkcalendar>' \
  https://server/rooms/conf-room-a.ics/
```

When invited, the room automatically accepts if available.

### Delegation

Delegate attendance to another user:

```bash
# Alice delegates to Dave
curl -X PUT \
  -u alice:password \
  -H "Content-Type: text/calendar" \
  -d 'BEGIN:VCALENDAR
VERSION:2.0
BEGIN:VEVENT
UID:meeting@example.com
DTSTART:20250120T140000Z
DTEND:20250120T150000Z
SUMMARY:Team Meeting
ORGANIZER:mailto:organizer@example.com
ATTENDEE;DELEGATED-TO="mailto:dave@example.com";PARTSTAT=DELEGATED:mailto:alice@example.com
ATTENDEE;DELEGATED-FROM="mailto:alice@example.com":mailto:dave@example.com
END:VEVENT
END:VCALENDAR' \
  https://server/alice/calendar.ics/meeting.ics
```

### Recurring Event Scheduling

Schedule recurring meetings with RECURRENCE-ID support:

```icalendar
BEGIN:VCALENDAR
VERSION:2.0
METHOD:REQUEST
BEGIN:VEVENT
UID:weekly-meeting@example.com
DTSTART:20250120T140000Z
DTEND:20250120T150000Z
RRULE:FREQ=WEEKLY;COUNT=10
SUMMARY:Weekly Team Meeting
ORGANIZER:mailto:alice@example.com
ATTENDEE;RSVP=TRUE:mailto:bob@example.com
SEQUENCE:0
END:VEVENT
END:VCALENDAR
```

Modify single instance:

```icalendar
BEGIN:VEVENT
UID:weekly-meeting@example.com
RECURRENCE-ID:20250127T140000Z
DTSTART:20250127T150000Z
DTEND:20250127T160000Z
SUMMARY:Weekly Team Meeting (Rescheduled)
ORGANIZER:mailto:alice@example.com
ATTENDEE:mailto:bob@example.com
SEQUENCE:1
END:VEVENT
```

### Free/Busy Queries

Check availability before scheduling:

```bash
curl -X REPORT \
  -u alice:password \
  -H "Content-Type: application/xml" \
  -d '<?xml version="1.0"?>
      <C:free-busy-query xmlns:C="urn:ietf:params:xml:ns:caldav">
        <C:time-range start="20250120T080000Z" end="20250120T170000Z"/>
      </C:free-busy-query>' \
  https://server/bob/calendar.ics/
```

Response includes free/busy periods:

```icalendar
BEGIN:VCALENDAR
VERSION:2.0
METHOD:REPLY
BEGIN:VFREEBUSY
UID:freebusy-query@example.com
DTSTART:20250120T080000Z
DTEND:20250120T170000Z
ORGANIZER:mailto:alice@example.com
ATTENDEE:mailto:bob@example.com
FREEBUSY;FBTYPE=BUSY:20250120T100000Z/20250120T110000Z
FREEBUSY;FBTYPE=BUSY:20250120T140000Z/20250120T150000Z
END:VFREEBUSY
END:VCALENDAR
```

## Configuration

Enable scheduling in Radicale:

```ini
[scheduling]
# Enable CalDAV scheduling
enabled = True

# Auto-accept resource calendars
resource_calendars = /rooms/, /equipment/

# Default schedule agent (SERVER or CLIENT)
default_schedule_agent = SERVER

# Allow organizer to see all attendee addresses
attendee_visibility = public
```
